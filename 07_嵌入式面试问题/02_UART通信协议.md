
# ? 一、UART 核心原理（必须掌握）

## 1?? 基本特点

* **异步通信**（无时钟线）
* 通过 **波特率约定 bit 时间**
* 使用 **起始位同步**

---

## 2?? 基本连接

```
TX → RX
RX → TX
GND → GND（必须）
```

---

## 3?? 电平规则

| 状态  | 电平  |
| --- | --- |
| 空闲  | 高   |
| 起始位 | 低   |
| 数据位 | 0/1 |
| 停止位 | 高   |

---

## 4?? 一帧结构

```
Start + Data + Parity(可选) + Stop
```

常见：

```
8N1 = 1 + 8 + 1 = 10 bit
```

---

## 5?? 发送顺序

```
Start → D0 → D1 → ... → D7 → Stop
```

? **LSB first（低位先发）**

---

# ? 二、时序与采样（核心理解）

## 1?? 采样流程

```
检测下降沿（起始位）
↓
等待 0.5 bit（确认）
↓
1.5 bit 采样 D0
↓
每 1 bit 采样一次
```

---

## 2?? 为什么中间采样？

* 避开边沿抖动
* 电平最稳定
* 抗干扰强

---

## 3?? 波特率与时间

```
bit时间 = 1 / 波特率
```

常用：

* 115200 → 8.68 μs
* 9600 → 104 μs

---

## 4?? 吞吐量

```
字节/s ≈ 波特率 / 10（8N1）
```

---

## 5?? 波特率误差本质

* 采样点逐 bit 偏移
* **误差累积**
* 总偏移不能超过 **0.5 bit**

---

# ?? 三、关键寄存器与错误

## 1?? RXNE

```
接收寄存器非空
```

清除：

```
读取 USART_DR
```

---

## 2?? ORE（溢出）

原因：

```
旧数据未读 + 新数据到来
```

清除：

```
读 SR → 读 DR
```

---

# ? 四、接收方式对比（工程重点）

## 1?? 查询（Polling）

* CPU一直检查
* 效率低

---

## 2?? 中断（RXNE）

* 每字节中断
* CPU负担大

---

## 3?? DMA

* 自动搬运数据
* 减少中断

---

# ? 五、DMA + IDLE（高频面试）

## 核心一句话

> DMA搬运数据，IDLE判断一帧结束

---

## 工作流程

```
数据 → DMA写内存
↓
数据停止
↓
IDLE触发
↓
计算长度
↓
处理数据
```

---

## 中断关键步骤

```
1. 读 SR
2. 读 DR（清IDLE）
3. 停止 DMA
4. 计算长度
5. 处理数据
6. 重启 DMA
```

---

## 长度计算

```
len = buffer_size - DMA_remaining
```

---

# ? 六、环形缓冲区（工程核心）

## 指针

* write_idx（DMA）
* read_idx（CPU）

---

## 数据量计算

### 情况1（无回绕）

```
write - read
```

### 情况2（回绕）

```
size - read + write
```

---

## 本质

> 用两个指针描述“数据流区间”

---


---

## 解析流程

```
找帧头 → 读长度 → 判断完整 → 校验 → 取数据
```

## 关键原则

# ? 八、工程设计原则（高分点）

## 1?? 中断原则

* 短小
* 只做搬运
* 不做解析

---

## 2?? 为什么不用 RXNE？

* 每字节中断
* CPU压力大

---

## 3?? 为什么用 DMA+IDLE？

* 一帧一次中断
* 高效稳定

# ? 最后一条“核心总结”（建议背）

> UART 是一种基于波特率的异步串行通信方式，通过起始位进行同步，在每个 bit 中间采样数据。工程中通常采用 DMA + IDLE + 环形缓冲区 + 协议解析的方案，实现高效、稳定的不定长数据接收。
